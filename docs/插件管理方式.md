# 插件管理方式说明

## 概述

插件系统支持两种管理方式：
1. **配置文件管理**（推荐）- 在 `configs/config.yaml` 中配置
2. **数据库管理** - 在数据库中存储插件状态

**优先级**：配置文件 > 数据库

## 配置文件管理（推荐）

### 配置方式

在 `configs/config.yaml` 中配置：

```yaml
plugins:
  dir: ./src/plugins
  auto_reload: false
  use_config_file: true  # 使用配置文件管理（true: 配置文件优先, false: 仅使用数据库）
  enabled:  # 启用的插件列表
    - example_plugin
    - hello
    - admin
  disabled:  # 禁用的插件列表（优先级高于enabled）
    # - example_plugin
```

### 配置说明

- **use_config_file**: 
  - `true` - 使用配置文件管理，配置文件优先于数据库
  - `false` - 仅使用数据库管理，忽略配置文件中的 enabled/disabled

- **enabled**: 启用的插件列表（插件名称，不含.py）
- **disabled**: 禁用的插件列表（优先级高于enabled）

### 示例

#### 启用多个插件

```yaml
plugins:
  enabled:
    - example_plugin
    - hello
    - admin
    - weather
```

#### 禁用特定插件

```yaml
plugins:
  enabled:
    - example_plugin
    - hello
    - admin
  disabled:
    - weather  # 即使在其他地方启用，也会被禁用
```

#### 启用所有插件（空列表表示全部启用）

```yaml
plugins:
  enabled: []  # 空列表表示启用所有插件
```

## 数据库管理

### 使用场景

- 需要动态启用/禁用插件（运行时修改）
- 需要持久化插件状态
- 需要为插件存储额外配置数据

### 启用插件

#### 方式 1: 使用 SQLite

```bash
sqlite3 data/bot.db "INSERT INTO plugin_configs (plugin_name, enabled, config_data) VALUES ('example_plugin', 1, '{}');"
```

#### 方式 2: 使用 Python 脚本

```python
import asyncio
from src.database.connection import DatabaseManager
from src.database.plugin_config import PluginConfig
from config import load_config

async def enable_plugin():
    config = load_config()
    db_manager = DatabaseManager(config.database)
    await db_manager.init_db()
    
    async with db_manager.get_session() as session:
        await PluginConfig.create_or_update(session, "example_plugin", enabled=True)
        print("插件已启用")

asyncio.run(enable_plugin())
```

### 禁用插件

```bash
sqlite3 data/bot.db "UPDATE plugin_configs SET enabled = 0 WHERE plugin_name = 'example_plugin';"
```

## 优先级说明

### 配置文件优先模式（use_config_file: true）

1. **首先**检查配置文件中的 `enabled` 列表
2. **然后**从 `enabled` 中移除 `disabled` 列表中的插件
3. **如果**配置文件为空，才从数据库读取

**示例**：
```yaml
plugins:
  use_config_file: true
  enabled:
    - example_plugin
  disabled:
    - hello
```

结果：只有 `example_plugin` 被启用，即使数据库中 `hello` 是启用的。

### 仅数据库模式（use_config_file: false）

1. **忽略**配置文件中的 `enabled` 和 `disabled`
2. **仅从**数据库读取插件状态

**示例**：
```yaml
plugins:
  use_config_file: false
  enabled:
    - example_plugin  # 这个会被忽略
```

结果：只从数据库读取插件状态。

## 使用建议

### 开发环境

推荐使用**配置文件管理**：
- ✅ 简单直观
- ✅ 易于版本控制
- ✅ 无需数据库操作

```yaml
plugins:
  use_config_file: true
  enabled:
    - example_plugin
    - hello
```

### 生产环境

可以选择：
1. **配置文件管理** - 适合固定配置
2. **数据库管理** - 适合需要动态调整的场景

### 混合使用

可以同时使用两种方式：
- 配置文件：定义默认启用的插件
- 数据库：运行时动态调整

设置 `use_config_file: true`，配置文件优先，数据库作为补充。

## 常见问题

### Q1: 插件没有启用？

**检查清单**：
1. 检查配置文件中的 `enabled` 列表
2. 检查 `disabled` 列表中是否包含该插件
3. 检查 `use_config_file` 设置
4. 如果使用数据库，检查数据库中的配置

### Q2: 如何查看当前启用的插件？

查看启动日志：
```
插件 example_plugin 已加载并启用
```

### Q3: 配置文件修改后需要重启吗？

**需要**。修改配置文件后需要重启机器人才能生效。

### Q4: 数据库和配置文件冲突怎么办？

配置文件优先。如果 `use_config_file: true`，配置文件中的设置会覆盖数据库。

### Q5: 如何临时禁用插件？

**方式 1**: 修改配置文件，添加到 `disabled` 列表
```yaml
plugins:
  disabled:
    - example_plugin
```

**方式 2**: 使用数据库（如果 use_config_file: false）
```bash
sqlite3 data/bot.db "UPDATE plugin_configs SET enabled = 0 WHERE plugin_name = 'example_plugin';"
```

## 配置示例

### 完整配置示例

```yaml
plugins:
  dir: ./src/plugins
  auto_reload: false
  use_config_file: true  # 使用配置文件管理
  
  # 启用的插件列表
  enabled:
    - example_plugin
    - hello
    - admin
  
  # 禁用的插件列表（优先级高于enabled）
  disabled:
    # - weather  # 注释表示不禁用
```

### 仅使用数据库

```yaml
plugins:
  dir: ./src/plugins
  auto_reload: false
  use_config_file: false  # 不使用配置文件，仅使用数据库
  enabled: []  # 会被忽略
  disabled: []  # 会被忽略
```

## 总结

- ✅ **配置文件管理**：简单、直观、适合开发环境
- ✅ **数据库管理**：灵活、动态、适合生产环境
- ✅ **混合使用**：配置文件优先，数据库补充
- ✅ **优先级**：配置文件 > 数据库

选择适合你的管理方式即可！
