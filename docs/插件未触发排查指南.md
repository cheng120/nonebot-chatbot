# 插件未触发排查指南

## 问题：`nonebot-plugin-anans-sketchbook` 插件未触发

### 可能原因

1. **插件未安装**
   - 检查插件是否已通过 pip 安装
   - 检查虚拟环境是否正确激活

2. **插件未加载**
   - 检查 `pyproject.toml` 中是否配置了插件
   - 检查 `configs/config.yaml` 中是否启用了插件
   - 查看启动日志，确认插件是否被加载

3. **配置问题**
   - API Key 未填写（`anan.api_key` 为空）
   - 配置项缺失或格式错误

4. **触发方式错误**
   - 插件的触发方式可能不是"夏目安安"
   - 需要查看插件文档确认正确的触发方式

### 排查步骤

#### 步骤 1: 检查插件是否已安装

```bash
cd /Users/cheng/Desktop/document/cheng/nonebot-chatbot
python3 -c "import nonebot_plugin_anans_sketchbook; print('插件已安装')"
```

如果报错 `ModuleNotFoundError`，说明插件未安装，需要安装：

```bash
pip install nonebot-plugin-anans-sketchbook
```

#### 步骤 2: 检查配置文件

检查 `configs/config.yaml`：

```yaml
plugins:
  enabled:
    - nonebot-plugin-anans-sketchbook  # 确保这一行存在
```

检查 `anan` 配置：

```yaml
anan:
  enabled: true
  api_key: 'your-api-key-here'  # 确保填写了 API Key
  api_base: https://api.anthropic.com
```

#### 步骤 3: 检查启动日志

重启机器人后，查看日志中是否有：

```
[INFO] 检测到 1 个外部插件: ['nonebot-plugin-anans-sketchbook']
[INFO] ✅ 已加载外部插件: nonebot-plugin-anans-sketchbook (模块: nonebot_plugin_anans_sketchbook)
[INFO] 插件 nonebot_plugin_anans_sketchbook 已加载并启用
```

如果没有这些日志，说明插件未正确加载。

#### 步骤 4: 检查插件的触发方式

`nonebot-plugin-anans-sketchbook` 的触发方式可能不是"夏目安安"。需要：

1. 查看插件文档或源代码
2. 确认正确的命令或关键词
3. 检查是否需要特定格式（如 `/命令` 或 `@机器人`）

#### 步骤 5: 使用 `/migrate_plugin` 命令自动迁移

如果插件未正确加载，可以使用迁移命令：

```
/migrate_plugin nonebot-plugin-anans-sketchbook
```

这个命令会：
1. 安装插件（如果未安装）
2. 自动复制到 `src/plugins/` 目录
3. 自动更新配置文件
4. 将外部插件转换为内部插件

### 常见问题

#### Q1: 插件已安装但未加载

**解决方案**：
1. 确保 `pyproject.toml` 中配置了插件
2. 确保 `configs/config.yaml` 中启用了插件
3. 重启机器人

#### Q2: API Key 未配置

**解决方案**：
1. 访问 [Anthropic Console](https://console.anthropic.com/) 获取 API Key
2. 在 `configs/config.yaml` 中填写 `anan.api_key`
3. 重启机器人

#### Q3: 触发方式不正确

**解决方案**：
1. 查看插件文档或 GitHub 仓库
2. 确认正确的命令格式
3. 尝试不同的触发方式（如 `/命令`、`@机器人`、直接发送消息等）

### 调试命令

在机器人运行时，可以查看已加载的插件：

```python
from nonebot import get_loaded_plugins
plugins = get_loaded_plugins()
for plugin in plugins:
    print(f"插件: {plugin.name}")
```
