# 消息日志功能说明

## 功能概述

消息日志功能会自动记录所有接收到的机器人消息，支持两种存储方式：
- **文件日志**：记录到 `logs/messages.log` 文件
- **数据库日志**：记录到数据库的 `message_logs` 表

## 功能特性

### 记录的信息

每条消息日志包含以下信息：

- **消息ID**：OneBot 消息唯一标识
- **消息类型**：`private`（私聊）或 `group`（群聊）
- **用户ID**：发送消息的用户ID
- **群组ID**：群消息的群组ID（仅群消息）
- **消息内容**：格式化后的消息文本
- **原始消息**：完整的消息对象（JSON格式）
- **发送者信息**：用户昵称、群名片等
- **时间戳**：消息接收时间

### 支持的消息类型

- ✅ 文本消息
- ✅ 图片消息
- ✅ 表情消息
- ✅ @消息
- ✅ 回复消息
- ✅ 文件消息
- ✅ 视频消息
- ✅ 语音消息

## 日志格式

### 文件日志格式

```
[2026-01-21T10:43:40] GROUP | 群123456 | 用户789012 | 这是一条群消息
[2026-01-21T10:43:41] PRIVATE | 用户789012 | 这是一条私聊消息
[2026-01-21T10:43:42] GROUP | 群123456 | 用户789012 | 这是一条带图片的消息 [图片: http://example.com/image.jpg]
```

### 数据库日志格式

数据库中的 `message_content` 字段存储 JSON 格式的详细信息：

```json
{
  "content": "这是一条消息 [图片: http://example.com/image.jpg]",
  "raw": "这是一条消息[图片:url=http://example.com/image.jpg]",
  "sender": {
    "user_id": 789012,
    "nickname": "用户昵称",
    "card": "群名片"
  }
}
```

## 配置

### 自动启用

消息日志功能在机器人启动时自动初始化，无需额外配置。

### 自定义配置

如果需要自定义配置，可以在 `bot.py` 中修改：

```python
# 初始化消息日志记录器
message_logger = init_message_logger(
	db_manager=db_manager,          # 数据库管理器（可选）
	log_to_file=True,                # 是否记录到文件（默认: True）
	log_file_path="./logs/messages.log"  # 日志文件路径（默认: ./logs/messages.log）
)
```

### 禁用数据库日志

如果不需要数据库日志，可以传入 `None`：

```python
message_logger = init_message_logger(
	db_manager=None,  # 禁用数据库日志
	log_to_file=True  # 只使用文件日志
)
```

## 查看日志

### 查看文件日志

```bash
# 实时查看日志
tail -f logs/messages.log

# 查看最近的日志
tail -n 100 logs/messages.log

# 搜索特定用户的消息
grep "用户123456" logs/messages.log

# 搜索特定群组的消息
grep "群789012" logs/messages.log
```

### 查看数据库日志

#### 使用 SQLite

```bash
sqlite3 data/bot.db

# 查看所有消息日志
SELECT * FROM message_logs ORDER BY created_at DESC LIMIT 10;

# 查看私聊消息
SELECT * FROM message_logs WHERE message_type = 'private' ORDER BY created_at DESC;

# 查看群消息
SELECT * FROM message_logs WHERE message_type = 'group' ORDER BY created_at DESC;

# 查看特定用户的消息
SELECT * FROM message_logs WHERE user_id = 123456 ORDER BY created_at DESC;

# 查看特定群组的消息
SELECT * FROM message_logs WHERE group_id = 789012 ORDER BY created_at DESC;
```

#### 使用 MySQL

```sql
-- 查看所有消息日志

-- 查看私聊消息
SELECT * FROM message_logs WHERE message_type = 'private' ORDER BY created_at DESC;

-- 查看群消息
SELECT * FROM message_logs WHERE message_type = 'group' ORDER BY created_at DESC;

-- 查看特定用户的消息
SELECT * FROM message_logs WHERE user_id = 123456 ORDER BY created_at DESC;

-- 查看特定群组的消息
SELECT * FROM message_logs WHERE group_id = 789012 ORDER BY created_at DESC;
```

## 日志轮转

文件日志支持自动轮转：

- **日志文件**：`logs/messages.log`
- **轮转方式**：按大小或时间自动轮转
- **旧日志**：自动压缩为 `.zip` 文件

## 性能考虑

### 文件日志

- ✅ 性能开销小
- ✅ 适合高频消息记录
- ✅ 日志文件会自动轮转

### 数据库日志

- ⚠️ 需要数据库连接
- ⚠️ 可能影响性能（高频消息时）
- ✅ 支持复杂查询
- ✅ 适合数据分析

### 建议

- **开发环境**：同时启用文件日志和数据库日志
- **生产环境**：建议只启用文件日志，或限制数据库日志的记录频率

## 清理日志

### 清理文件日志

```bash
# 删除旧日志文件
rm logs/messages.log.*

# 清空当前日志文件
> logs/messages.log
```

### 清理数据库日志

```sql
-- 删除所有日志（谨慎操作）
DELETE FROM message_logs;

-- 删除7天前的日志
DELETE FROM message_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 7 DAY);

-- 删除特定用户的消息日志
DELETE FROM message_logs WHERE user_id = 123456;
```

## 故障排查

### 日志未记录

1. **检查日志文件权限**
   ```bash
   ls -l logs/messages.log
   ```

2. **检查数据库连接**
   ```bash
   sqlite3 data/bot.db "SELECT COUNT(*) FROM message_logs;"
   ```

3. **查看错误日志**
   ```bash
   grep "消息日志" logs/bot.log
   ```

### 日志文件过大

1. **启用日志轮转**（已默认启用）
2. **定期清理旧日志**
3. **考虑禁用数据库日志**（如果不需要）

## API 使用

### 在代码中使用消息日志

```python
from src.services.message_logger import get_message_logger

# 获取消息日志记录器
message_logger = get_message_logger()

# 手动记录消息（通常不需要，消息处理器会自动记录）
if message_logger:
    await message_logger.log_message(event)
```
