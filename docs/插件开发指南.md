## 插件系统概述

### 插件目录

插件默认存放在 `src/plugins/` 目录下。插件管理器会自动扫描该目录下的所有 `.py` 文件（排除 `__init__.py` 和以 `_` 开头的文件）。

### 插件加载机制

1. **自动扫描**：启动时自动扫描 `src/plugins/` 目录
2. **数据库管理**：插件状态（启用/禁用）存储在数据库中
3. **动态加载**：只加载数据库中标记为"启用"的插件

### 插件命名规则

- 插件文件名：`plugin_name.py`（小写字母、下划线）
- 插件名称：NoneBot 会自动从文件名生成（如 `hello.py` → `hello`）

## 创建新插件

### 步骤 1: 创建插件文件

在 `src/plugins/` 目录下创建新的 Python 文件，例如 `my_plugin.py`：

```python
"""
我的插件
插件描述信息
"""
from nonebot import on_command, on_message
from nonebot.adapters.onebot.v11 import Bot, MessageEvent, Message
from src.utils.logger import get_logger

logger = get_logger("my_plugin")

# 创建命令处理器
my_command = on_command("mycmd", priority=10, block=True)

@my_command.handle()
async def handle_my_command(event: MessageEvent):
    """处理命令"""
    logger.info(f"用户 {event.user_id} 使用了 mycmd 命令")
    await my_command.send("这是我的插件响应！")
```

### 步骤 2: 启用插件

插件创建后，需要在数据库中启用它。有两种方式：

#### 方式 1: 通过代码启用（开发时）

```python
from src.services.plugin_manager import PluginManager

# 假设已有 plugin_manager 实例
await plugin_manager.enable_plugin("my_plugin")
```

#### 方式 2: 通过数据库直接启用

插件首次加载时，如果数据库中没有配置，会自动标记为"禁用"。需要手动在数据库中启用：

```sql
-- 假设使用 SQLite
INSERT INTO plugin_configs (plugin_name, enabled, config_data) 
VALUES ('my_plugin', 1, '{}');
```

### 步骤 3: 重启机器人

重启机器人后，插件会自动加载并启用。

## 插件开发规范

### 1. 基本结构

```python
"""
插件名称
插件描述
"""
from nonebot import on_command, on_message, on_notice, on_request
from nonebot.adapters.onebot.v11 import Bot, MessageEvent, Message
from src.utils.logger import get_logger

logger = get_logger("plugin_name")

# 定义处理器
handler = on_command("command_name", priority=10, block=True)

@handler.handle()
async def handle_function(event: MessageEvent):
    """处理函数"""
    # 插件逻辑
    pass
```

### 2. 常用处理器类型

#### 命令处理器

```python
from nonebot import on_command

# 创建命令处理器
cmd = on_command("命令名", priority=10, block=True)

@cmd.handle()
async def handle_cmd(event: MessageEvent):
    await cmd.send("响应消息")
```

#### 消息处理器

```python
from nonebot import on_message
from nonebot.rule import to_me

# 创建消息处理器（@机器人时触发）
msg = on_message(priority=10, block=True, rule=to_me())

@msg.handle()
async def handle_msg(event: MessageEvent):
    await msg.send("响应消息")
```

#### 通知处理器

```python
from nonebot import on_notice
from nonebot.adapters.onebot.v11 import NoticeEvent

# 创建通知处理器
notice = on_notice(priority=10)

@notice.handle()
async def handle_notice(event: NoticeEvent):
    # 处理通知事件（如群成员加入、离开等）
    pass
```

#### 请求处理器

```python
from nonebot import on_request
from nonebot.adapters.onebot.v11 import RequestEvent

# 创建请求处理器
request = on_request(priority=10)

@request.handle()
async def handle_request(event: RequestEvent):
    # 处理请求事件（如好友申请、群申请等）
    pass
```

### 3. 日志记录

使用项目统一的日志系统：

```python
from src.utils.logger import get_logger

logger = get_logger("plugin_name")

logger.info("信息日志")
logger.warning("警告日志")
logger.error("错误日志")
logger.debug("调试日志")
```

### 4. 错误处理

```python
@handler.handle()
async def handle_function(event: MessageEvent):
    try:
        # 插件逻辑
        pass
    except Exception as e:
        logger.error(f"处理失败: {e}", exc_info=True)
        await handler.send("处理失败，请稍后重试")
```

### 5. 参数解析

```python
from nonebot.params import CommandArg

@cmd.handle()
async def handle_cmd(event: MessageEvent, args: Message = CommandArg()):
    # args 是命令后的参数
    arg_str = str(args).strip()
    if arg_str:
        await cmd.send(f"收到参数: {arg_str}")
    else:
        await cmd.send("请提供参数")
```

### 6. 权限控制

```python
from nonebot.permission import SUPERUSER

# 只允许超级用户使用
admin_cmd = on_command("admin", permission=SUPERUSER, priority=10, block=True)

@admin_cmd.handle()
async def handle_admin(event: MessageEvent):
    await admin_cmd.send("这是管理员命令")
```

### 7. 定时任务

```python
from nonebot import require
from nonebot_plugin_apscheduler import scheduler

# 需要安装 nonebot-plugin-apscheduler
require("nonebot_plugin_apscheduler")

@scheduler.scheduled_job("cron", hour=0, minute=0)
async def daily_task():
    """每天 00:00 执行"""
    logger.info("执行定时任务")
```

## 插件配置管理

### 读取配置

插件可以通过 `PluginManager` 读取自己的配置：

```python
from src.services.plugin_manager import PluginManager

# 假设在插件中获取配置
# 注意：需要从 bot.py 中获取 plugin_manager 实例
# 或者通过依赖注入获取

async def get_my_config():
    # 通过某种方式获取 plugin_manager
    config = await plugin_manager.get_plugin_config("my_plugin")
    if config:
        return config.get("config_data", {})
    return {}
```

### 更新配置

```python
# 更新插件配置
config_data = {
    "key1": "value1",
    "key2": "value2"
}
await plugin_manager.update_plugin_config("my_plugin", config_data)
```

## 插件启用/禁用

### 启用插件

```python
await plugin_manager.enable_plugin("plugin_name")
```

### 禁用插件

```python
await plugin_manager.disable_plugin("plugin_name")
```

### 查看插件状态

```python
config = await plugin_manager.get_plugin_config("plugin_name")
if config:
    enabled = config.get("enabled", False)
    logger.info(f"插件状态: {'启用' if enabled else '禁用'}")
```

### 示例 4: 完整功能插件模板

```python
"""
完整插件模板
包含命令、消息、配置等功能
"""
from nonebot import on_command, on_message
from nonebot.adapters.onebot.v11 import Bot, MessageEvent, Message
from nonebot.params import CommandArg
from nonebot.rule import to_me
from src.utils.logger import get_logger

logger = get_logger("template_plugin")

# ========== 命令处理器 ==========
hello_cmd = on_command("hello", priority=10, block=True)

@hello_cmd.handle()
async def handle_hello(event: MessageEvent):
    """打招呼命令"""
    user_id = event.user_id
    logger.info(f"用户 {user_id} 使用了 hello 命令")
    await hello_cmd.send(f"你好，用户 {user_id}！")

# ========== 消息处理器 ==========
at_handler = on_message(priority=10, block=True, rule=to_me())

@at_handler.handle()
async def handle_at_me(event: MessageEvent):
    """处理@机器人的消息"""
    message = str(event.message).strip()
    logger.info(f"收到@消息: {message}")
    await at_handler.send(f"你说了: {message}")

# ========== 带参数的命令 ==========
echo_cmd = on_command("echo", priority=10, block=True)

@echo_cmd.handle()
async def handle_echo(event: MessageEvent, args: Message = CommandArg()):
    """回显命令"""
    arg_str = str(args).strip()
    if arg_str:
        await echo_cmd.send(f"回显: {arg_str}")
    else:
        await echo_cmd.send("用法: /echo <内容>")

# ========== 插件初始化（可选） ==========
def _init():
    """插件初始化"""
    logger.info("模板插件已加载")

_init()
```

## 常见问题

### Q1: 插件创建后没有生效？

**A:** 检查以下几点：
1. 插件文件是否在 `src/plugins/` 目录下
2. 插件文件名是否符合规范（不能以 `_` 开头，不能是 `__init__.py`）
3. 插件是否已在数据库中启用
4. 查看日志是否有错误信息

### Q2: 如何调试插件？

**A:** 
1. 查看日志文件：`logs/` 目录
2. 使用 `logger.debug()` 输出调试信息
3. 检查插件是否正确加载：查看启动日志中的"插件加载完成"信息

### Q3: 插件如何访问数据库？

**A:** 
```python

# 或者通过依赖注入
from src.database.connection import DatabaseManager

async def use_database():
    # 假设有 db_manager 实例
    async with db_manager.get_session() as session:
        # 使用 session 进行数据库操作
        pass
```

### Q4: 插件如何读取配置？

**A:** 
```python
# 方式1: 从 PluginManager 读取
config = await plugin_manager.get_plugin_config("plugin_name")

# 方式2: 从配置文件读取（需要实现配置加载逻辑）
from src.utils.config_loader import load_config
config = load_config()
```

**A:** 目前插件系统不支持热重载，需要重启机器人。未来可能会添加此功能。
