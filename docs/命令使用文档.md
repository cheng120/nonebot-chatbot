## 命令前缀说明

### 命令格式

机器人支持以下命令格式：

- **无前缀命令**：直接输入命令名（如 `hello`）
- **斜杠前缀**：使用 `/` 前缀（如 `/hello`）
- **@机器人**：直接 @机器人并发送消息

### 命令分隔符

- 空格：`/echo 你好世界`
- 点号：`/echo.你好世界`

## 示例插件命令

示例插件 (`example_plugin`) 提供了多个实用功能，包含以下命令：

### 1. 打招呼命令

**命令**: `hello` / `你好` / `hi`

**说明**: 与机器人打招呼，机器人会友好地回应。

**示例**:
```
hello
你好
hi
```

**响应**:
```
你好，用户 123456！欢迎使用 NoneBot 机器人！
```

---

### 2. 回显命令

**命令**: `/echo <内容>`

**说明**: 机器人会回显你发送的内容。

**参数**:
- `<内容>`: 要回显的文本内容（必需）

**示例**:
```
/echo 你好世界
/echo 这是一条测试消息
```

**响应**:
```
📢 回显: 你好世界
```

**错误提示**:
如果未提供内容，机器人会提示：
```
❌ 用法: /echo <内容>
示例: /echo 你好世界
```

---

### 3. 计算器命令

**命令**: `/calc <表达式>` / `/计算 <表达式>` / `/计算器 <表达式>`

**说明**: 简单的数学计算器，支持基本数学运算。

**参数**:
- `<表达式>`: 数学表达式（必需）

**示例**:
```
/calc 1+2*3
/calc (10+5)/3
/计算 2^3
```

**响应**:
```
📊 计算结果: 1+2*3 = 7
```

**错误提示**:
如果表达式无效，机器人会提示错误信息。

**注意**: 此计算器仅支持基本运算，不支持复杂函数。

---

### 4. 机器人信息命令

**命令**: `/info` / `/信息` / `/状态`

**说明**: 显示机器人的基本信息，包括插件名称、当前时间、用户ID等。

**示例**:
```
/info
```

**响应**:
```
🤖 机器人信息

• 插件名称: 示例插件 (example_plugin)
• 当前时间: 2026-01-21 20:00:00
• 用户ID: 123456
• 消息类型: group

这是一个功能完整的插件示例！
```

---

### 5. 图片发送命令

**命令**: `/image [URL]` / `/图片 [URL]`

**说明**: 发送图片。可以发送指定URL的图片，或不带参数时发送示例图片。

**参数**:
- `[URL]`: 图片URL（可选）

**示例**:
```
/image https://example.com/image.jpg
/image
/图片
```

**响应**:
- 带URL时：发送指定URL的图片
- 不带URL时：发送示例占位图，并提示：
  ```
  💡 提示: 使用 /image <图片URL> 可以发送指定图片
  ```

**错误提示**:
如果图片URL无效或发送失败，机器人会提示错误信息。

---

### 6. 今日人品命令

**命令**: `/jrrp` / `/今日人品` / `/人品` / `/rp`

**说明**: 查看今日人品值（0-100）。基于用户ID和日期生成，同一天同一用户的人品值相同。

**示例**:
```
/jrrp
/今日人品
/人品
```

**响应**:
```
🌟 你的今日人品：85/100

运气不错！

💡 提示：人品值每天都会更新，同一天内保持不变
```

**人品值说明**:
- **90-100**: 🌟 大吉大利！
- **80-89**: ✨ 运气不错！
- **70-79**: 😊 还算可以
- **60-69**: 🙂 平平淡淡
- **50-59**: 😐 一般般
- **40-49**: 😕 有点倒霉
- **30-39**: 😟 不太顺利
- **20-29**: 😰 运气很差
- **10-19**: 😱 非常糟糕
- **0-9**: 💀 极度不幸

---

### 7. 插件迁移命令

**命令**: `/migrate_plugin <插件名>` / `/迁移插件 <插件名>` / `/安装插件 <插件名>`

**说明**: 安装并分析市场插件，便于将功能移植到 `example_plugin` 中。

**参数**:
- `<插件名>`: 要迁移的插件名称（必需），如 `nonebot-plugin-jrrp3`

**示例**:
```
/migrate_plugin nonebot-plugin-jrrp3
/迁移插件 nonebot-plugin-epicfree
/安装插件 nonebot-plugin-xxx
```

**响应**:
```
🔄 开始迁移插件: nonebot-plugin-jrrp3
请稍候...

✅ 插件迁移完成: nonebot-plugin-jrrp3

📦 提取的信息:
• 命令数量: 1
• 函数数量: 3
• 导入语句: 5 条

💡 提示: 插件已安装，但功能代码需要手动集成到 example_plugin.py 中
💡 建议: 查看插件源代码，手动复制需要的功能代码

📝 插件位置: /path/to/plugin/source.py
```

**功能说明**:
1. 自动使用 `pip` 安装指定插件
2. 查找插件的 Python 模块路径
3. 读取并分析插件源代码
4. 提取插件中的命令、函数和导入语句
5. 生成详细的迁移报告

**注意事项**:
- 需要网络连接以下载插件
- 插件会安装到当前 Python 环境
- 功能代码需要手动集成到 `example_plugin.py`
- 建议根据报告手动复制需要的功能代码

**错误提示**:
- 如果插件名称错误或不存在，会提示安装失败
- 如果无法找到插件模块，会提示迁移失败
- 如果插件中没有可迁移的内容，会提示警告

## 消息交互

### @机器人

直接 @机器人并发送消息，机器人会响应。

**示例**:
```
@机器人 帮助
@机器人 help
```

**响应**:
- 如果消息包含"帮助"或"help"，机器人会显示帮助信息：
  ```
  🤖 示例插件帮助

  可用命令：
  • hello - 打招呼
  • /echo <内容> - 回显内容
  • /calc <表达式> - 简单计算器
  • /info - 显示机器人信息
  • /jrrp 或 /今日人品 - 查看今日人品值
  • /migrate_plugin <插件名> - 安装并迁移市场插件

  直接@我也可以和我对话哦！
  ```

- 其他消息会收到友好回复：
  ```
  👋 你@我了！我收到了你的消息：xxx

  输入「帮助」查看可用命令
  ```

## CLI 工具命令

项目提供了命令行工具 `nb-add-plugin`，用于快速创建插件模板。

### 安装

```bash
# 从源码安装（开发模式）
cd nonebot-chatbot
pip install -e .

# 或从 PyPI 安装
pip install nonebot-chatbot
```

### 基本用法

```bash
# 创建最简单的插件
nb-add-plugin my_plugin

# 创建带描述的插件
nb-add-plugin weather --description "天气查询插件"

# 创建带使用说明的插件
nb-add-plugin admin --description "管理员插件" --usage "/admin - 管理员命令"
```

### 完整参数

```bash
nb-add-plugin <插件名称> [选项]
```

**必需参数**:
- `name`: 插件名称（文件名，不含.py）

**可选参数**:
- `-d, --description`: 插件描述（默认: 插件名称）
- `-u, --usage`: 使用说明（默认: 自动生成）
- `--dir`: 插件目录（默认: `src/plugins`）
- `--author`: 作者（默认: `NoneBot Chatbot Team`）
- `--homepage`: 主页链接（默认: 项目主页）
- `-f, --force`: 覆盖已存在的文件

### 示例

#### 创建简单插件

```bash
nb-add-plugin hello
```

生成文件: `src/plugins/hello.py`

#### 创建带描述的插件

```bash
nb-add-plugin weather --description "天气查询插件，支持查询全国主要城市天气"
```

#### 创建带使用说明的插件

```bash
nb-add-plugin calculator \
  --description "计算器插件" \
  --usage "/calc <表达式> - 计算数学表达式\n示例: /calc 1+2*3"
```

更多信息请参考 [CLI工具使用指南](./CLI工具使用指南.md)。

## 常见问题

### Q1: 命令没有响应？

**可能原因**:
1. 插件未启用：检查 `configs/config.yaml` 中的 `plugins.enabled` 列表
2. 命令前缀错误：尝试使用 `/` 前缀或直接输入命令名
3. 机器人未连接：检查机器人连接状态

**解决方法**:
1. 确认插件在启用列表中
2. 尝试不同的命令格式（`hello` 或 `/hello`）
3. 检查机器人日志，查看是否有错误信息

---

### Q2: 如何添加新命令？

**方法1: 使用 CLI 工具**（推荐）

```bash
nb-add-plugin my_command
```

**方法2: 手动创建**

1. 在 `src/plugins/` 目录下创建新的插件文件
2. 参考 `example_plugin.py` 的代码结构
3. 在 `configs/config.yaml` 的 `plugins.enabled` 中添加插件名称

更多信息请参考 [插件开发指南](./插件开发指南.md)。

---

### Q3: 如何迁移市场插件？

使用 `/migrate_plugin` 命令：

```
/migrate_plugin nonebot-plugin-xxx
```

命令会自动：
1. 安装插件
2. 分析插件代码
3. 生成迁移报告

然后根据报告手动集成功能代码。

---

### Q4: 命令优先级如何工作？

命令处理器的优先级（`priority`）决定了处理顺序：
- **高优先级**（如 `priority=10`）：优先处理
- **低优先级**（如 `priority=1`）：后处理

示例插件中的命令都使用 `priority=10`，确保优先处理。

---

### Q5: 如何查看所有可用命令？

**方法1**: @机器人并发送"帮助"或"help"

**方法2**: 查看本文档

**方法3**: 查看插件源代码中的 `__plugin_meta__.usage`
