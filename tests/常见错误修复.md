# 常见测试错误修复指南

## 已修复的常见问题

### 1. 数据库模型定义问题

**问题**: SQLAlchemy Column的comment参数在某些版本不支持

**修复**: 移除Column定义中的comment参数，使用文档字符串代替

**文件**: 
- `src/database/models.py`
- `src/database/plugin_config.py`

---

### 2. 数据库初始化方式

**问题**: 只使用SQL脚本可能在某些情况下失败

**修复**: 
- 优先使用ORM模型创建表（`Base.metadata.create_all`）
- 如果失败，回退到SQL迁移脚本

**文件**: `src/database/connection.py`

---

### 3. 路径比较问题

**问题**: Path对象比较可能因格式不同而失败

**修复**: 同时比较Path对象和字符串表示

**文件**: `tests/test_services.py`

---

### 4. models.py导入问题

**问题**: 导入了未使用的 `declarative_base`

**修复**: 移除未使用的导入

**文件**: `src/database/models.py`

---

## 如果测试仍然失败

### 步骤1: 运行调试脚本

```bash
cd nonebot-chatbot
python3 debug_tests.py
```

这会显示详细的错误信息。

### 步骤2: 检查具体错误

根据错误信息，常见问题：

#### A. 导入错误

**错误**: `ModuleNotFoundError: No module named 'xxx'`

**解决**:
```bash
# 确保在项目根目录
cd nonebot-chatbot

# 激活虚拟环境
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

#### B. 数据库连接错误

**错误**: `sqlalchemy.exc.OperationalError`

**解决**:
- 检查数据库路径权限
- 确保目录存在
- 检查SQLite文件是否被锁定

#### C. 异步测试错误

**错误**: `RuntimeError: Event loop is closed`

**解决**:
- 确保安装了 `pytest-asyncio`
- 检查 `pytest.ini` 中的 `asyncio_mode = auto`

#### D. 配置加载错误

**错误**: `pydantic.ValidationError`

**解决**:
- 检查配置文件格式
- 使用 `reset_config()` 重置全局配置
- 检查环境变量设置
