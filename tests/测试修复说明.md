# 测试修复说明

## 已修复的问题

### 1. 数据库模块问题

**问题**: `plugin_config.py` 中使用了 `datetime.now()` 但未导入

**修复**: 在文件开头添加 `from datetime import datetime`

**文件**: `src/database/plugin_config.py`

---

### 2. 配置单例测试问题

**问题**: 测试配置单例时，全局配置可能已被其他测试加载

**修复**: 
- 在 `config.py` 中添加 `reset_config()` 函数用于测试
- 在测试中使用 `reset_config()` 重置全局配置

**文件**: 
- `config.py` - 添加 `reset_config()` 和 `force_reload` 参数
- `tests/test_config.py` - 使用 `reset_config()` 重置配置

---

### 3. 数据库会话测试问题

**问题**: 某些测试在使用数据库会话前未初始化数据库

**修复**: 在测试中添加数据库初始化步骤

**文件**: `tests/test_database.py`
- `test_get_session`: 添加 `await db_manager.init_db()`

---

### 4. 消息处理测试问题

**问题**: `MessageSegment` 的API使用不正确

**修复**: 修正 `MessageSegment` 的参数格式
- `MessageSegment.image(file="...")` 而不是 `MessageSegment.image("...")`
- `MessageSegment.at(qq=123456)` 而不是 `MessageSegment.at(123456)`
- `MessageSegment.reply(id=12345)` 而不是 `MessageSegment.reply(12345)`

**文件**: 
- `tests/test_message_handler.py` - 修正所有MessageSegment调用
- `src/services/message_handler.py` - 改进类型转换处理

---

### 5. 插件管理器测试问题

**问题**: 路径比较可能因格式不同而失败

**修复**: 使用 `Path.resolve()` 进行路径比较

**文件**: `tests/test_services.py`

---

### 6. 集成测试清理问题

**问题**: 测试中数据库连接可能未正确清理

**修复**: 使用 `try-finally` 确保资源清理

**文件**: `tests/test_integration.py`

---

## 测试执行建议

### 运行所有测试

```bash
cd nonebot-chatbot
source venv/bin/activate  # 如果使用虚拟环境
pytest tests/ -v
```

### 运行特定测试模块

```bash
# 配置测试
pytest tests/test_config.py -v

# 数据库测试
pytest tests/test_database.py -v

# 工具模块测试
pytest tests/test_utils.py -v
```

### 生成覆盖率报告

```bash
pytest tests/ --cov=src --cov=config --cov-report=html --cov-report=term
```

---

## 已知限制

1. **适配器测试**: `test_adapters.py` 中的适配器设置测试需要NoneBot运行环境，在没有环境时会跳过
2. **消息处理测试**: 需要NoneBot适配器库，某些测试可能因API版本差异而需要调整
3. **插件加载测试**: 需要实际的插件文件，测试中使用临时插件目录

---

## 如果测试仍然失败

1. **检查依赖**: 确保所有依赖已安装
   ```bash
   pip install -r requirements.txt
   ```

2. **检查虚拟环境**: 确保虚拟环境已激活
   ```bash
   source venv/bin/activate
   ```

3. **查看详细错误**: 使用 `-v -s` 参数查看详细输出
   ```bash
   pytest tests/ -v -s
   ```

4. **检查Python版本**: 确保使用Python 3.9+
   ```bash
   python --version
   ```
